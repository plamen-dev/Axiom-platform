# Plan Review Queue

## Purpose

Governed human-review layer for plans generated by the
Knowledge-Aware Capability Planner (PR #45).

Axiom can generate structured plans, but there is no durable mechanism
to decide whether a plan is approved, rejected, deferred, needs more
evidence, or superseded.  This module creates that decision layer.

## Non-Goals

- No plan execution
- No automatic approval
- No promotion
- No learning
- No workflow mutation

## Architecture

```
┌───────────────────────────────┐
│  Capability Planner (PR #45)  │
│  Generates structured plans   │
└──────────────┬────────────────┘
               │ plan_id
               ▼
┌───────────────────────────────┐
│     Plan Review Registry      │
│                               │
│  • PlanReview                 │
│  • PlanReviewDecision         │
│  • PlanReviewStatus           │
│  • PlanReviewEvidence         │
│  • PlanReviewHistory          │
│                               │
│  Persistence: SQLite          │
└──────────────┬────────────────┘
               │
               ▼
┌───────────────────────────────┐
│         CLI Surface           │
│                               │
│  axiom plan-reviews           │
│  axiom plan-review            │
│  axiom plan-review-create     │
└───────────────────────────────┘
```

## Inputs (Consumed by Reference)

| Source | What |
|--------|------|
| PR #45 Capability Planner | Plan IDs, plan names, plan structure |
| Knowledge Reviews | Review decisions (cross-reference) |
| Capability State | Promotion history |
| Evidence Bundles | Validation artifacts |
| Failure Classifications | Risk context |

## Decisions

| Decision | Meaning |
|----------|---------|
| `proposed` | Plan created, awaiting human review |
| `approved` | Plan trusted to move forward |
| `rejected` | Plan refused (safety, policy, evidence) |
| `deferred` | Plan postponed for later consideration |
| `needs_more_evidence` | Plan requires additional validation |
| `superseded` | Plan replaced by a newer version |

## Reasons

| Reason | When Used |
|--------|-----------|
| `human_validation` | Human reviewed and decided |
| `insufficient_evidence` | Not enough data to decide |
| `unsafe` | Plan involves unvalidated mutations |
| `conflicting_plan` | Another plan contradicts this one |
| `duplicate` | Same plan already exists |
| `obsolete` | Plan no longer relevant |
| `low_confidence` | Planner confidence below threshold |
| `founder_override` | Explicit founder decision |
| `policy_violation` | Plan violates governance rules |
| `deferred_pending` | Awaiting prerequisite completion |

## CLI Commands

### `axiom plan-reviews`

List all plan review records with optional filters.

```bash
axiom plan-reviews
axiom plan-reviews --decision approved
axiom plan-reviews --status open
axiom plan-reviews --plan-id <id>
axiom plan-reviews --json-output
```

### `axiom plan-review --plan-id <id>`

Show detailed review information for a specific plan.

```bash
axiom plan-review --plan-id plan-abc123
axiom plan-review --plan-id plan-abc123 --json-output
```

### `axiom plan-review-create`

Create a new review decision for a plan.

```bash
axiom plan-review-create --plan-id plan-abc123 --decision approved --reason human_validation
axiom plan-review-create --plan-id plan-abc123 --decision rejected --reason unsafe --notes "Unvalidated mutation"
```

## Persistence

- Table: `plan_reviews` — review records
- Table: `plan_review_events` — lifecycle event log (history)
- SQLAlchemy ORM with SQLite backend (WAL mode)
- Reuses existing Axiom database layer

## History and Supersession

Multiple reviews for the same plan are preserved as full history.
`PlanReviewHistory` provides the complete chain with latest decision.
Supersession links old → new reviews via `superseded_by` field.

## Strategic Purpose

- PR #45 creates plans (capability planning)
- PR #51 decides which plans are trusted enough to move forward (this module)
- Future: approved plans feed into execution scheduling
